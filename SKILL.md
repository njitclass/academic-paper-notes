# 顶刊学术论文学习笔记

## 工作原则：批判性视角贯穿始终

生成笔记的每个模块时，同步追问以下四个问题，发现的问题积累到模块6集中呈现：

- **假设是否过强？** 识别策略前提是否成立（平行趋势、IV外生性、系统边界合理性）
- **论证是否跳跃？** 从数据/模型到结论之间是否有未经检验的逻辑环节
- **结果是否被设计？** 参数选取、样本范围、边界划定是否系统性有利于结论
- **外部效度如何？** 结论能否推广到其他情境、国家、行业、时间段

顶刊文章通过同行评审并不意味着无懈可击——发现实质性缺陷本身就是高价值的研究切入口。

## 步骤0：接收输入

### PDF输入
用户上传PDF时，调用内置 **pdf skill** 提取全文，不自行编写提取脚本。

### 文字输入
用户粘贴摘要或全文时，直接使用，并在笔记开头标注"输入完整度：摘要/部分/全文"。仅有摘要时，在信息不足的字段标注`[摘要未提供，推测]`。

### 批量输入判断
- 用户提供**多篇**内容时 → 进入**批量模式**，执行步骤4的循环逻辑
- 用户提供**单篇**内容时 → 进入**单篇精读模式**

### 学科迁移方向确认（必须在开始生成笔记前执行）

接收到输入后，**立即**向用户展示以下选择清单，等待回复后再开始生成任何模块：

```
📚 请选择你希望在【方法论迁移地图】中展开的学科方向（可多选，回复序号即可）：

  1. 能源工程经济学  — LCOE/系统优化/LCA/情景分析方向
  2. 能源政策/计量  — DID/IV/面板数据/政策评估方向
  3. 管理科学/营销  — 博弈论/运营管理/消费者行为方向
  4. MPaCC/会计     — 碳信息披露/成本分析/ESG审计方向

  例如：回复 "1 3" 表示展开方向1和3；回复 "全部" 表示四个方向都展开。
```

收到回复后，记录用户选择的方向，**仅对所选方向**在模块6中展开迁移建议，其余方向略去。

## 步骤1：判断方法论路线

根据期刊名和论文关键词，判断走哪条路线：

| 路线 | 触发期刊/特征 |
|------|--------------|
| **模式A**（计量实证） | Energy Economics, Energy Policy, TAR, JAE, CAR, JAR, AER, QJE, RES, JPE, Marketing Science, JMR, AMJ, AMR；关键词：regression, DID, instrumental variable, fixed effects, panel data |
| **模式B**（工程经济学） | Nature Energy, Applied Energy, Energy Conversion and Management, Renewable Energy, Joule；关键词：optimization, LCOE, LCA, energy system, robust optimization, game theory, scenario analysis |
| **模式A+B**（混合型） | Applied Energy等期刊中同时含计量回归和系统优化/LCA的文章 |

判断模糊时：加载 `references/journal-profiles.md` 查询，仍不确定则在笔记开头声明推断结果并请用户确认。

**确定路线后，再执行步骤2。**

### 模块3：🖼️ 图表导读

> **触发条件**：仅在 PDF 全文输入时执行完整图表分析。
> 文字/摘要输入时，标注 `[图表导读跳过 — 请提供 PDF 全文以启用本模块]`，然后继续模块4。

阅读方法和结果之前，先建立"图表全景地图"：扫描全文，识别 **3–5 张关键图** 和 **2–3 张关键表**，其余可略过。这一步的价值在于让读者在深入细节之前先有视觉锚点——后续模块4/5 引用图表时读者知道该去哪里看原文。

**关键图的识别优先级**：
- 方法架构图：系统框架图、模型流程图、识别策略图（DID 事件研究图、IV 路径图等）
- 核心结果图：主系数图、Pareto 前沿图、情景比较图
- 机制/稳健性图：中介效应图、敏感性分析图、异质性分析图

**关键表的识别优先级**：
- 主回归/基准结果表（通常是 Table 2 或 Table 3）
- 描述性统计表（含关键变量分布信息）
- 成本/参数汇总表（模式B论文）

---

**格式规范（严格遵守）**：

1. **编号与 caption 必须与原文完全一致**，包括子图（Fig. 3a/3b）和补充材料图（Fig. S1）——不得自行重新编号或改写
2. **保留原文 caption 完整英文原文**，不翻译、不缩减
3. 图统一用 **Fig. N** 格式，表统一用 **Table N** 格式（Table 不缩写为 Tab.）
4. **PDF 中图/表的 caption 若无法可靠读取，必须标注** `[原文 caption 无法读取 — 请参见原文第 X 页 Fig./Table N]`，**严禁根据图表内容自行推断或编造标题**。PDF 排版中 caption 常与图像交织，文本提取器容易错位，此时宁可留白也不可编造。

---

**每张关键图，按以下格式输出**：

> **[Fig. N]** *（原文完整 caption，英文原文）*
> `📷 图片占位符 — 原文第 X 页`
>
> **图解**：该图展示了什么，核心视觉信息是什么（1–2句）
> **关键性**：为什么这张图对理解论文至关重要，与哪个核心论点直接挂钩
> **批判性观察**：图中是否有异常、缺失的对比组、置信区间是否被省略等（如无则略）

**每张关键表，按以下格式输出**：

> **[Table N]** *（原文完整 caption，英文原文）*
>
> | 原文列名1 | 原文列名2 | … |
> |-----------|-----------|---|
> | 核心数值行1（原文数据） | … | … |
> | 核心数值行2（原文数据） | … | … |
>
> *(仅重现 2–4 行最关键数据，完整表见原文。列名保持英文原文。)*
>
> **表解**：该表的核心发现，量化结论（1–2句）
> **批判性观察**：是否缺少标准误/p值、对照组是否完整等（如无则略）

---

> 后续模块4/5 在涉及具体图表时，在相关句末标注 `→ 见 Fig. N` 或 `→ 见 Table N`，帮助读者快速定位原文对应位置。

### 模块4：📐 数据与研究方法

**按路线展开——** 模式A见 `references/method-econometrics.md` 模板，模式B见 `references/method-engineering-econ.md` 模板，模式A+B两套并列输出，各自标注`[计量部分]`和`[工程建模部分]`。引用具体图表时在句末标注 `→ 见 Fig. N` 或 `→ 见 Table N`（编号来自模块3）。

此模块是全文最重要的模块，要求：

- **公式必须使用原文的符号、下标和结构，一个不能省略或替换。** 使用你当前平台最擅长的方式读取公式（视觉识别、原生PDF解析工具、文本提取均可），不限定技术路径。公式识别后用 LaTeX 写出，不用文字代替。
- **若公式无法可靠读取，标注** `[公式无法读取 — 请参见原文第 X 页 Eq. N]`，**严禁自行推导或编造替代公式。** PDF 中的数学公式常以图像形式嵌入，文本提取结果极易错位；无法确认时宁可标注也不可编造。
- **原文有编号的公式，用 `\tag{Eq. N}` 保留原文编号**（原文括号式如 `(3)` 统一写为 `\tag{Eq. 3}`），使读者能按编号在原文中快速定位。
- **公式无法读取时，紧随其后的符号说明（nomenclature）通常为纯文本，提取成功率更高——应仍原样保留呈现**，不因公式缺失而连同略去。
- **每个可读公式之后必须紧跟符号说明**，逐行列出原文中每个符号的含义、单位（如适用）及取值范围（如适用），不得出现无说明的裸公式。
- 识别策略/约束方程须完整，不可省略关键条件
- 参数来源须注明（实测/文献/假设），为批判性评估积累素材

## 步骤3：写入文件

### 文件命名
```
note-{第一作者姓}-{发表年份}-{标题前5实词，空格→_}.md
```
实词定义：跳过冠词(a/an/the)、介词(of/in/for/on等)、连词。

### 文件写入要求（所有平台强制执行）

生成笔记内容后，**必须使用平台原生文件写入工具将完整8模块内容写入磁盘**，
不得仅向对话流输出。

各平台写入方式：
- **Claude Code**：使用 `Write` 工具写入文件
- **Codex CLI / Gemini CLI**：使用 Python `open().write()` 工具调用，
  或支持文件操作的原生工具
- **其他平台**：使用该平台提供的文件写入 API / 工具调用

写入后必须：
1. 输出文件**绝对路径**供用户验证
2. 确认文件存在（如平台支持，调用文件读取工具验证）

> ⚠️ **严禁将完整8模块笔记内容输出到对话流。禁止使用 bash heredoc 模板
> （单引号定界符禁止变量展开，无法注入 Agent 生成的内容）。**

### 单篇精读模式 / 批量模式（输出格式统一）

## 步骤4：批量循环逻辑

### 对话模式（Cowork / 不支持文件系统访问的平台）

用户提交N篇时，**立即计算并告知**：
```
📋 检测到N篇论文，需N轮处理（每轮1篇）。
   本轮：第1篇 → 完成后请继续提交第2篇。
```

每轮处理1篇，完成后提示下一轮所需操作。循环由用户手动触发。

### Agent 自动执行模式（支持文件系统与工具调用的平台）

用户指定项目文件夹时，执行以下逻辑：

```bash
# 1. 扫描所有PDF，计算总数
PDF_FILES=($(ls *.pdf 2>/dev/null))
TOTAL=${#PDF_FILES[@]}

if [ $TOTAL -eq 0 ]; then
    echo "❌ 当前目录未发现PDF文件，请 cd 到正确目录后重试"
    exit 1
fi

echo "📋 共发现${TOTAL}篇PDF，开始逐篇处理"
echo "📁 输出目录：$(pwd)"

# 2. 逐篇循环（每次1篇）
for ((i=0; i<TOTAL; i++)); do
    PDF="${PDF_FILES[$i]}"
    echo "--- 第$((i+1))/${TOTAL}轮：${PDF} ---"

    # [调用pdf skill提取全文 → 执行步骤1-3]
    # [FILENAME 由步骤3命名规则生成：note-{作者姓}-{年份}-{标题前5实词}.md]
    # [使用步骤3"文件写入要求"中的平台原生写入工具写入文件]

    echo "✅ [$((i+1))/${TOTAL}] ${FILENAME} 已写入 $(pwd)"
done
```
