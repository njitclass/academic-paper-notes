# 顶刊学术论文学习笔记

## 工作原则：批判性视角贯穿始终

生成笔记的每个模块时，同步追问以下四个问题，发现的问题积累到模块6集中呈现：

- **假设是否过强？** 识别策略前提是否成立（平行趋势、IV外生性、系统边界合理性）
- **论证是否跳跃？** 从数据/模型到结论之间是否有未经检验的逻辑环节
- **结果是否被设计？** 参数选取、样本范围、边界划定是否系统性有利于结论
- **外部效度如何？** 结论能否推广到其他情境、国家、行业、时间段

顶刊文章通过同行评审并不意味着无懈可击——发现实质性缺陷本身就是高价值的研究切入口。

---

## 步骤0：接收输入

### PDF输入
用户上传PDF时，调用内置 **pdf skill** 提取全文，不自行编写提取脚本。

### 文字输入
用户粘贴摘要或全文时，直接使用，并在笔记开头标注"输入完整度：摘要/部分/全文"。仅有摘要时，在信息不足的字段标注`[摘要未提供，推测]`。

### 批量输入判断
- 用户提供**多篇**内容时 → 进入**批量模式**，执行步骤4的循环逻辑
- 用户提供**单篇**内容时 → 进入**单篇精读模式**

---

## 步骤1：判断方法论路线

根据期刊名和论文关键词，判断走哪条路线：

| 路线 | 触发期刊/特征 |
|------|--------------|
| **模式A**（计量实证） | Energy Economics, Energy Policy, TAR, JAE, CAR, JAR, AER, QJE, RES, JPE, Marketing Science, JMR, AMJ, AMR；关键词：regression, DID, instrumental variable, fixed effects, panel data |
| **模式B**（工程经济学） | Nature Energy, Applied Energy, Energy Conversion and Management, Renewable Energy, Joule；关键词：optimization, LCOE, LCA, energy system, robust optimization, game theory, scenario analysis |
| **模式A+B**（混合型） | Applied Energy等期刊中同时含计量回归和系统优化/LCA的文章 |

判断模糊时：加载 `references/journal-profiles.md` 查询，仍不确定则在笔记开头声明推断结果并请用户确认。

**确定路线后，再执行步骤2。**

## 步骤2：生成7模块笔记

> **按需加载规则（在生成对应模块前执行）：**
> - 模式含A → 生成模块3/4前，读取 `references/method-econometrics.md`
> - 模式含B → 生成模块3/4前，读取 `references/method-engineering-econ.md`
> - 生成模块5前 → 读取 `references/transfer-guide.md`
> - 上述加载后即可生成，不需要预先全部加载

### 模块3：📐 数据与研究方法

**按路线展开——** 模式A见 `references/method-econometrics.md` 模板，模式B见 `references/method-engineering-econ.md` 模板，模式A+B两套并列输出，各自标注`[计量部分]`和`[工程建模部分]`。

此模块是全文最重要的模块，要求：
- 所有公式用LaTeX写出，不用文字代替
- **每个公式之后必须紧跟符号说明**，逐行列出每个符号的含义、单位（如适用）及典型取值范围（如适用），不得出现无说明的裸公式
- 识别策略/约束方程须完整，不可省略关键条件
- 参数来源须注明（实测/文献/假设），为批判性评估积累素材

---

### 模块4：📊 主要结果与检验

**按路线展开——** 同模块3，分别按 `references/method-econometrics.md` 和 `references/method-engineering-econ.md` 中的结果模板呈现。

核心要求：
- 结论须量化（系数值/LCOE数值/优化目标值），不接受纯文字描述
- 检验/敏感性分析须列出清单，注明通过/未通过

## 步骤3：写入文件

### 文件命名
```
note-{第一作者姓}-{发表年份}-{标题前5实词，空格→_}.md
```
实词定义：跳过冠词(a/an/the)、介词(of/in/for/on等)、连词。
示例：`note-Zhang-2024-Carbon_trading_scheme_optimal_design.md`

### 单篇精读模式 / 批量模式（输出格式统一）

两种模式均采用相同输出策略：
- 笔记**写入 `.md` 文件**（完整7模块内容）
- 对话中只输出：**文件链接 + 约400字结构化摘要**，格式：

```
✅ [X/N] note-Zhang-2024-... 已生成（单篇时省略进度标记）
[查看完整笔记](computer://.../.md)
└─ 核心问题：...
└─ 方法：[模式A/B/AB] | [识别策略/核心模型，1句]
└─ 主要结论：[量化结论，2-3句]
└─ 最强切入口：[1个最具价值的研究切入，含目标期刊]
└─ 迁移亮点：[1个最可操作的迁移建议]
└─ 顶刊亮点：[1个值得借鉴的写作/方法呈现亮点]
```

摘要设计保证后续追问无需重新读取文件；如需查看详情，直接打开文件链接。

## 步骤4：批量循环逻辑

### Cowork 环境

用户提交N篇时，**立即计算并告知**：
```
📋 检测到N篇论文，需N轮处理（每轮1篇）。
   本轮：第1篇 → 完成后请继续提交第2篇。
```

每轮处理1篇，完成后提示下一轮所需操作。循环由用户手动触发。

### Claude Code 环境

用户指定项目文件夹时，执行以下逻辑：

```bash
# 1. 扫描所有PDF，计算总数
PDF_FILES=($(ls *.pdf 2>/dev/null))
TOTAL=${#PDF_FILES[@]}
echo "📋 共发现${TOTAL}篇PDF，逐篇处理"

# 2. 逐篇循环（每次1篇）
for ((i=0; i<TOTAL; i++)); do
    echo "--- 第$((i+1))/${TOTAL}轮 ---"
    # [调用pdf skill提取 → 执行步骤1-3 → 写入文件]
    echo "✅ [$((i+1))/${TOTAL}] note-... 已生成"
done

echo "🎉 全部完成：共${TOTAL}篇，生成${TOTAL}个.md文件"
```

全部完成后，在对话末尾追加**跨文比较摘要**（方法论异同 + 可组合的研究方向），不单独生成文件。
