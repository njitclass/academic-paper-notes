## 一、项目背景与定位

### 1.1 用户背景

| 维度 | 说明 |
|------|------|
| 角色 | 商学院教授 + MPaCC 硕士导师 |
| 研究专长 | **能源工程经济学**、管理科学、营销学 |
| 能源方向细化 | 偏工程经济学方法（技术经济评价、系统优化、能源规划、博弈模型、鲁棒优化），而非计量经济学路线；顶刊为 Nature Energy、Applied Energy；不使用 HOMER/TIMES 等商业规划软件，自行编程建模 |
| 新兼任方向 | 会计/MPaCC（非本专业，需借助研究方法迁移） |
| 学生群体 | 研究型研究生（方法基础强）+ MPaCC硕士（方法基础薄弱） |

### 1.2 核心痛点

1. **方法论迁移障碍**：跨领域使用计量、实验、文本分析等方法时，缺少系统化的提取工具
2. **研究灵感触发低效**：读完一篇论文后，难以快速识别可延伸的研究方向
3. **学生指导成本高**：不同基础的学生需要反复解释同一篇文章的核心方法，缺乏标准化笔记
4. **跨领域文献整合难**：能源/管理/营销/会计四个方向的文献逻辑差异大，难以统一分析框架

### 1.3 Skill 定位

> 一个面向商学院研究者的**顶刊论文深度学习笔记生成器**，核心价值在于：
> **方法论精准提取 × 跨领域迁移建议 × 研究灵感触发**

## 二、使用场景

### 2.1 主要使用目的（由浅入深）

1. 深度理解核心概念与理论框架（读懂这篇文章在说什么）
2. 解析方法论细节，掌握可复制的研究设计（读懂这篇文章怎么做的）
3. 批判性评估：识别论文的逻辑缺陷、假设漏洞、边界局限（判断这篇文章哪里做得不够好）
4. **将缺陷转化为研究切入口**：发现"我能做得比这篇更严谨/更完整"的选题逻辑
5. 方法论迁移：将本文设计移植到自己/学生的研究领域
6. 寻找研究灵感与选题延伸方向
7. 撰写文献综述（提取 lit review 素材与评述角度）
8. 学习顶刊写作规范与论证结构

### 2.2 适用期刊范围

| 领域 | 代表期刊 | 主流方法论路线 |
|------|----------|----------------|
| **能源工程经济学**（核心方向） | **Nature Energy, Applied Energy**, Energy Conversion and Management, Renewable Energy | 技术经济评价、系统优化模型、生命周期分析（LCA）、能源系统规划、情景分析 |
| 能源政策/计量经济 | Energy Economics, Energy Policy | 计量识别策略、政策评估、DID/IV |
| 综合管理/营销 | Marketing Science, JMR, AMJ, AMR | 实验设计、结构方程、计量识别 |
| 会计/审计/公司治理 | TAR, JAE, CAR, JAR | 计量识别、事件研究、文本分析 |
| 经济学综合 | AER, QJE, RES, JPE | 理论模型、计量识别 |

### 2.3 典型触发场景

- "帮我深度解读这篇论文"
- "分析这篇 Nature Energy 的系统优化模型"
- "这篇 Applied Energy 的 LCA 方法能用在会计的碳排放核算方向吗？"
- "把这篇文章整理成可以给学生看的笔记"
- "这篇能源系统规划的情景分析框架，能迁移到企业碳中和战略研究吗？"
- "分析这篇 Energy Economics 的识别策略"
- "这篇 DID 的设计我想复刻，帮我拆解"

## 三、功能需求

### 3.1 输入规格

| 输入方式 | 说明 |
|----------|------|
| PDF 上传 | 直接上传论文 PDF 文件 |
| 文字粘贴 | 粘贴摘要、全文或部分章节 |
| 两种并存 | 根据情况灵活选择，Skill 均需支持 |

### 3.2 输出规格

| 维度 | 要求 |
|------|------|
| 格式 | 结构化 Markdown，支持 LaTeX 公式（$...$ 和 $$...$$） |
| 文件 | 每篇独立 .md 文件，直接写入磁盘 |
| 深度定位 | 始终输出教授级别深度，学生自行决定追问程度 |

**单篇精读模式 / 批量模式（输出格式统一）**：

两种模式均采用相同策略——笔记完整写入 .md 文件，对话中只输出文件链接 + 约400字结构化摘要，格式如下：

```
✅ note-Zhang-2024-... 已生成（批量时加进度标记 [X/N]）
[查看完整笔记](computer://.../.md)
└─ 核心问题：...
└─ 方法：[模式A/B/AB] | [识别策略/核心模型，1句]
└─ 主要结论：[量化结论，2-3句]
└─ 最强切入口：[1个最具价值的研究切入，含目标期刊]
└─ 迁移亮点：[1个最可操作的迁移建议]
└─ 顶刊亮点：[1个值得借鉴的写作/方法呈现亮点]
```

设计动因：笔记全文篇幅通常超过 300 行，在对话框中全量展示会触发上下文压缩，反而影响后续追问质量；摘要足以支撑后续追问，详情通过文件链接直接查看，节省约 70–80% 输出 token。

### 3.3 笔记模板结构（7大模块）

#### 模块 1：题录信息

```
📌 题录信息
- APA格式完整引用
- 期刊名称 + 影响因子档位（Q1/Q2/顶刊）
- DOI
- 数据可得性（是否公开数据集）
- 代码可得性（是否提供复现代码）
- 发表年份 + 论文类型（实证/理论/综述）
```

#### 模块 2：研究问题与理论贡献

```
🎯 研究问题与理论贡献
- 核心研究问题（一句话）
- 理论机制（为什么X导致Y）
- 对既有文献的增量贡献（填补了什么空白）
- 理论框架/模型（如适用，含LaTeX公式）
- 关键假设
```

#### 模块 3：数据与研究方法（★核心模块★）

> **注意**：该模块需根据论文所属方法论路线自动切换解析框架，分为两种模式：

**模式A：计量/实证经济学路线**（适用于 Energy Economics、TAR、JAE、AER 等）
```
📐 数据与实证策略
- 数据来源与样本描述（时间范围、国家/行业、观测数量）
- 核心变量测量方式（被解释变量、解释变量、控制变量）
- 识别策略（内生性问题 + 解决方案）
  - 因果识别工具（DID/PSM/IV/RDD/断点回归等）
  - 工具变量选取逻辑（如适用）
  - 平行趋势检验（如DID）
- 基准回归模型（LaTeX公式）
- 固定效应设置（个体/时间/行业等）
- 标准误处理（聚类方式）
```

**模式B：工程经济学/系统优化路线**（适用于 Nature Energy、Applied Energy、Energy Conversion and Management 等）
```
📐 研究方法与模型架构
- 研究系统边界与范围界定（地理范围、时间跨度、技术路线）
- 核心方法论类型（从下列中识别）：
  □ 技术经济评价（LCOE/NPV/IRR/CAPEX-OPEX分析）
  □ 能源系统优化模型（线性规划/混合整数规划/多目标优化）
  □ 生命周期分析 LCA（系统边界、功能单元、影响类别）
  □ 情景分析与蒙特卡洛模拟（参数假设、不确定性范围）
  □ 能源规划模型（TIMES/OSeMOSYS/HOMER等工具）
  □ 数据驱动/机器学习方法（预测模型、需求侧建模）
- **物质与能量平衡约束**（须显式列出）：
  - 能量守恒方程（输入/输出/损耗）
  - 物质流约束（质量守恒、碳/氢/氧元素平衡，如适用）
  - 功率/容量约束（装机上限、爬坡速率、储能充放电约束等）
  - 系统耦合约束（多能源载体耦合、网络潮流约束等）
- 关键参数与假设（折现率、技术寿命、能源价格路径等）
- 模型核心公式（LaTeX）
- 敏感性/鲁棒性分析设计（哪些参数做了敏感性测试，是否采用鲁棒优化框架）
- 数据来源（技术参数库、统计数据库、实测数据）
```

#### 模块 4：主要结果与稳健性/敏感性检验

> 同样区分两种路线：

**模式A：计量路线**
```
📊 主要结果与稳健性检验
- 核心回归结果（主要系数 + 显著性）
- 经济显著性（效应量大小）
- 异质性分析（分样本/分组）
- 稳健性检验清单：
  □ 替换核心变量测量
  □ 安慰剂检验
  □ 子样本稳健性
  □ 其他方法稳健性
- 机制检验（中介/调节分析）
```

**模式B：工程经济学路线**
```
📊 主要结论与敏感性分析
- 核心量化结论（如：LCOE = X $/MWh，较基准降低Y%）
- 关键驱动因素排序（哪些参数对结果影响最大）
- 情景对比结论（不同政策/技术路径下的结果差异）
- 敏感性分析结果：
  □ 单因素敏感性（龙卷风图/蜘蛛图）
  □ 蒙特卡洛概率分布
  □ 情景分析边界值
- 与现有文献/基准案例的对比
- 政策含义与技术路线建议
```

#### 模块 5：方法论迁移地图（★核心创新模块★）

```
🔬 方法论迁移地图
- 本文核心方法论亮点（1-3个可迁移的设计）
- 迁移建议（具体化到用户研究领域）：
  → 能源工程经济学方向（Nature Energy/Applied Energy路线）：
     [如：该LCOE框架可用于中国分布式光伏的投资决策分析]
  → 能源政策/计量方向（Energy Economics路线）：
     [如：该情景分析框架可结合DID评估政策冲击效果]
  → 管理科学/营销方向：[具体课题建议]
  → MPaCC/会计方向：
     [如：LCA方法论可迁移至碳排放信息披露的可信度研究]
- 迁移前提条件（数据可得性、技术参数可及性、场景匹配度）
- 方法局限性提示（哪些情境不适合直接迁移）
- 工具/软件可及性（如涉及TIMES/HOMER等专业软件，说明替代方案）
```

#### 模块 6：批判性评估

```
🔍 批判性评估与研究切入口
- 方法论局限性：
  □ 关键假设是否过强或未充分检验（须具体指出哪个假设、为何存疑）
  □ 识别策略/模型设定的潜在缺陷（如平行趋势可疑、工具变量相关性不足、系统边界选取偏向）
  □ 样本选择偏差/外部效度限制
- 理论局限性（论证逻辑跳跃、机制黑箱）
- 顶刊审稿视角：这篇文章哪里做得特别出色（反向学习写作规范）
- **缺陷→研究切入口**（每个实质性缺陷须对应一个切入方向）：
  □ 缺陷描述 → 可改进的具体研究问题 → 适合的期刊方向
  示例："本文仅考虑光伏系统，未纳入储能耦合 →
        切入：含储能的综合能源系统LCOE优化 →
        期刊：Applied Energy / Energy Conversion and Management"
```

#### 模块 7：研究灵感笔记

```
💡 研究灵感笔记
- 本文直接启发的研究问题（3-5个）
- 可延伸的数据/场景（中国情境/其他行业/其他国家）
- 与当前研究热点的交叉点
- 导师建议（适合MPaCC学生的入门切入点）
```

### 3.5 公式符号说明规范

**所有 LaTeX 公式之后必须紧跟符号说明表**，不得仅呈现裸公式。要求：

- 每个出现在公式中的符号均须逐行说明其含义、单位（如适用）及取值范围或典型值（如适用）
- 格式采用紧凑表格或缩进列表（视符号数量而定），示例：

```
其中：
- $\text{LCOH}$：平准化制氢成本（$/kg H₂）
- $C_{\text{CapEx}}$：资本支出（$/kW），含设备采购与安装
- $C_{\text{OpEx}}$：年运营维护成本（$/kW/年）
- $\text{WACC}$：加权平均资本成本（%），反映融资风险溢价
- $H_{2,n}$：第 $n$ 年氢产量（kg），由电解槽效率与可再生出力决定
- $N$：项目评估期（年，通常取 20–25 年）
```

- 对于从原文直接提取的符号，优先使用原文定义；原文未定义的中间变量须补充说明
- 符号说明是笔记方法论可复用性的基础，不可省略

### 3.6 方法论重点覆盖

Skill 需要能精准识别和解析以下研究设计类型：

**路线A：计量/实证经济学方法**

| 类型 | 具体方法 |
|------|----------|
| 因果推断/识别策略 | DID、PSM、IV、RDD、断点回归、事件研究 |
| 面板数据与固定效应 | 双向固定效应、动态面板、异质性分析 |
| 不确定性与敏感性分析 | 安慰剂检验、Bootstrap、贝叶斯估计 |
| 文本分析/机器学习 | NLP、情感分析、主题模型、大模型标注 |
| 理论模型构建 | 博弈论、优化模型、分析模型 |

**路线B：工程经济学/能源系统方法**（新增，对应 Nature Energy / Applied Energy）

| 类型 | 具体方法 |
|------|----------|
| 技术经济评价 | LCOE、NPV、IRR、CAPEX/OPEX分解、平准化成本分析 |
| 能源系统优化 | 线性规划（LP）、混合整数规划（MILP）、多目标优化（Pareto前沿） |
| 生命周期分析 | LCA（ISO 14040/44）、碳足迹核算、全生命周期成本 |
| 情景与不确定性 | 情景分析（悲观/基准/乐观）、蒙特卡洛模拟、龙卷风图 |
| 系统最优化建模 | LP/MILP/多目标优化（Pareto前沿）、自编程实现（Python/Julia/MATLAB） |
| 博弈论模型 | Nash均衡、Stackelberg博弈、合作博弈、机制设计 |
| 鲁棒优化与不确定性 | 鲁棒优化（RO）、机会约束规划、区间分析、信息间隙决策理论（IGDT） |
| 数据驱动方法 | 需求预测、负荷建模、机器学习预测、混合物理-数据模型 |

## 四、非功能需求

### 4.1 输出质量标准

- 所有方法论解析须**精确、可操作**，而非泛泛概括
- LaTeX公式须正确对应原文模型
- 迁移建议须**具体到课题级别**，不接受"可以用于管理研究"这类模糊表述
- 批判性评估须从顶刊审稿人视角出发，有实质内容

### 4.4 批判性视角作为底层工作原则

批判性思维不仅是模块6的一个输出项，而是贯穿全文分析的底层视角。在解析每个模块时，Claude应同步追问：

- **假设是否过强？** 识别策略的前提条件是否在本文情境中成立（如平行趋势、工具变量外生性、系统边界合理性）
- **论证是否有跳跃？** 从数据到结论之间是否存在未经检验的逻辑环节
- **参数/样本是否有利于结论？** 是否存在选择性呈现结果的可能
- **外部效度如何？** 结论能否推广到其他情境、国家、时间段

发现的缺陷须在模块6中明确标注**"潜在研究切入口"**，说明如何将该缺陷转化为可独立发表的研究问题。这是该Skill区别于普通文献阅读工具的核心价值之一。

### 4.2 交互规范

- 默认全量输出所有7个模块
- 用户可追问：
  - "详细展开[模块名]"
  - "给我解释[方法]的直觉原理"（面向学生视角）
  - "这个识别策略的具体步骤是什么"
  - "帮我写一段 lit review 的引用段落"
- Skill 应在笔记末尾提示可用的追问方向

### 4.3 适应性要求

- **期刊领域自适应**：根据期刊类型自动切换方法论解析框架——
  - Nature Energy / Applied Energy / Energy Conversion and Management → 启用工程经济学模式（模式B）
  - Energy Economics / Energy Policy → 识别是计量路线还是工程经济路线后选择
  - TAR / JAE / AER 等 → 启用计量实证模式（模式A）
  - 无法判断时：先推断，再在笔记开头声明所用模式并提示用户确认
- **方法复杂度自适应**：高复杂度方法（如结构方程、机器学习）自动增加解析深度
- **输入完整度容错**：仅有摘要时，明确标注哪些模块信息不足，并给出推测性分析

## 六、Skill 文件结构规划

## 七、成功标准

读完一篇顶刊论文后，该 Skill 生成的笔记应能：

1. **让教授在 5 分钟内**判断该方法是否可迁移到自己或学生的课题
2. **让 MPaCC 学生在追问后**理解该识别策略的基本直觉（无需看原文）
3. **直接产出** 3 个以上具体、可操作的研究延伸方向
4. **作为 lit review 素材**，摘录模块 2 和模块 4 后可直接嵌入综述写作
5. **识别研究空白**，帮助判断该课题是否值得深入跟进

## 八、批量输出规格

### 8.1 触发方式

用户一次提供多篇论文（PDF上传或文字粘贴）时，自动进入批量模式。

### 8.2 上下文与循环策略

每批固定处理 **1篇**（单篇笔记约300行，叠加PDF全文输入后已占用大量上下文；每批2篇风险触发token吞吐上限，影响稳定性）。

**Cowork环境**：
- 用户提交N篇时，开始前自动计算并告知：`📋 共N篇，需N轮，本轮处理第1篇`
- 每篇完成时实时输出进度行 + 400字摘要：`✅ [1/N] note-Zhang-2024-Carbon_trading... 已生成`
- 每轮结束提示：`第1轮完成（1/N）。请继续提交第2篇。`
- 无法跨对话自动持续，每轮由用户手动触发下一篇

**Claude Code环境**（能力更强）：
- 用bash自动扫描项目文件夹中所有PDF，统计总数
- 在同一会话中自动逐篇循环处理，**无需用户手动触发**
- 进度展示格式：`[2/6] ✅ note-Li-2023-Robust_optimization_energy... 已生成`
- 全部完成后输出汇总：`🎉 共处理6篇，生成6个.md文件`

### 8.3 文件命名与导出

每篇笔记单独导出为 `.md` 文件，命名规则：

```
note-{第一作者姓}-{发表年份}-{文章标题（前5实词，空格替换为_）}.md
```

示例：`note-Zhang-2024-Carbon_trading_scheme_optimal_design.md`

- 若为混合型文章（Applied Energy 等），文件名不变，但笔记内部模块3/4标注 `[模式A+B]`
- 文章标题截取英文原题前5个实词（跳过冠词 a/an/the）
- 两篇均完成后，在对话末尾附加**跨文比较摘要**（方法论异同、可组合方向），不单独生成文件

## 九、按需加载架构

### 9.1 设计原则

SKILL.md常驻上下文（目标≤400行），references/按需读入，为论文全文保留最大上下文空间。

### 9.2 加载规则

| 文件 | 加载时机 |
|------|----------|
| `SKILL.md` | 始终加载（含模式判断逻辑、7模块骨架、批量流程） |
| `references/method-econometrics.md` | 仅识别为模式A时加载 |
| `references/method-engineering-econ.md` | 仅识别为模式B时加载（含约束方程模板） |
| `references/transfer-guide.md` | 生成模块5（迁移地图）时加载 |
| `references/journal-profiles.md` | 期刊判断模糊时按需加载，其余时候跳过 |
| pdf skill（内置） | 用户上传PDF时，直接调用已加载的pdf skill提取全文，无需自备脚本 |

### 9.3 Cowork环境限制说明

- **子代理（agents/）**：Cowork不支持，不使用
- **Zotero MCP**：Cowork无此连接，不使用
- **scripts/脚本**：本skill不设scripts/，PDF提取通过内置pdf skill完成，更健壮
- **批量循环**：无法跨对话自动持续，每轮由用户手动触发，Claude负责计算进度并告知
